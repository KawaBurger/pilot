# Pilot

A web-based remote control for [Claude Code](https://docs.anthropic.com/en/docs/claude-code) sessions. Create, resume, and interact with Claude Code conversations from your browser — anywhere on your Tailscale network.

Pilot runs Claude Code inside tmux sessions on your dev machine, watches Claude's JSONL conversation files for changes, and pushes updates to the browser via SSE.

## How it works

1. Pilot starts an HTTP server on your machine
2. You open the web UI from any device on your Tailscale network
3. Create a new session or resume an existing one — Pilot launches Claude Code in a tmux session behind the scenes
4. Messages are sent to Claude via tmux, and conversation updates stream back to the browser in real time by watching Claude's JSONL files

## Prerequisites

- [Go](https://go.dev/) 1.23+
- [Node.js](https://nodejs.org/) 18+ (for building the frontend)
- [tmux](https://github.com/tmux/tmux) — `brew install tmux`
- [Claude Code](https://docs.anthropic.com/en/docs/claude-code) CLI — `npm install -g @anthropic-ai/claude-code`
- [Tailscale](https://tailscale.com/) — for secure remote access from other devices

### Session titles

Session titles are displayed in the web UI but are **not generated by Pilot itself**. Pilot reads title files from `~/.claude/session-titles/{sessionID}`, which are created by a separate Claude Code stop hook.

To enable session titles, install [claude-session-title](https://github.com/KawaBurger/claude-session-title). Without it, Pilot falls back to showing the first message of each conversation as the title.

## Installation

```shell
git clone https://github.com/KawaBurger/pilot.git
cd pilot
make build
```

This builds the React frontend and compiles the Go binary.

## Setup

### 1. Create a user account

```shell
./pilot admin create <username>
```

You'll be prompted to set a password.

### 2. (Optional) Configure

Create `~/.pilot/config.yaml` to override defaults:

```yaml
server:
  host: "127.0.0.1"   # default; set to "0.0.0.0" to listen on all interfaces
  port: 8090
  allowed_origins:
    - "http://localhost:8090"
    - "http://your-machine.tailnet-name.ts.net:8090"

defaults:
  cwd: "/path/to/your/projects"   # default working directory for new sessions

claude:
  home: "~/.claude"   # Claude Code data directory
```

### 3. (Optional) Custom slash commands

Create `~/.pilot/commands.yaml`:

```yaml
commands:
  - name: review
    label: Review
    template: "review the recent changes"
  - name: refactor
    label: Refactor
    template: "refactor this code"
```

## Usage

```shell
# Start the server
./pilot serve

# Or run in background
make deploy

# Stop background server
make stop
```

Open `http://localhost:8090` (or `http://your-machine.tailnet-name.ts.net:8090` from another device on your Tailscale network).

## Tailscale setup

Pilot is designed to be accessed remotely via [Tailscale](https://tailscale.com/):

1. Install Tailscale on your dev machine and the device you want to access Pilot from
2. Both devices join the same tailnet
3. Set `allowed_origins` in config to include your Tailscale hostname
4. Access Pilot at `http://<tailscale-hostname>:8090`

No port forwarding or public exposure needed — Tailscale provides encrypted point-to-point connections within your private network.
